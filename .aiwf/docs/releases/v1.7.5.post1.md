# AIWF v1.7.5.post1

## Release Type

Metadata UX hotfix for v1.7.5.

## Changed

- Added a centralized metadata allowed-values registry.
- Added `./aiwf metadata allowed-values`.
- Expanded known provider and tool values.
- Added `reasoning_effort` validation.
- Improved metadata validation errors with allowed-values guidance.
- Updated metadata defaults to be provider/tool-neutral for `metadata init`.
- Updated metadata profile creation defaults to produce profile-backed metadata.
- Clarified source and confidence semantics in operator-facing docs.

## Compatibility

- No workflow protocol semantic change.
- No event schema change.
- No metadata migration.
- No `v1.7.6` bump.
