# AIWF v1.7.5.post2

## Release Type

Patch release for metadata attribution correctness.

## Changed

- Changed default task metadata attribution from Codex-specific values to neutral workflow roles:
  - `owner: "ai-agent"`
  - `reviewer: "human"`
- Clarified that `owner` and `reviewer` are workflow role fields, not provider/model provenance fields.
- Preserved compatibility for historical finalized records using `owner: "codex"` or `reviewer: "codex"`.
- Bumped `AIWF_TOOL_VERSION` to `1.7.5.post2` so event provenance matches the shipped attribution patch.

## Compatibility

- `WORKFLOW_PROTOCOL_VERSION` remains `1.7.5`.
- No workflow protocol semantic change.
- No event schema change.
- No historical record migration.
- No strict phase state machine change.
