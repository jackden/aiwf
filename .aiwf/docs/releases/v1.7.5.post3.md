# AIWF v1.7.5.post3

## Release Type

Metadata visibility and profile runtime option hardening.

## Changed

- Added `./aiwf metadata profile show [name]`.
- Enhanced `./aiwf metadata show` to include source layer/path for effective metadata fields.
- Added `./aiwf metadata status` for full metadata resolution visibility.
- `./aiwf metadata profile create <name>` now writes `AIWF_EVENT_LOG=1` by default.
- Active-profile runtime option resolution now includes `AIWF_EVENT_LOG` from profile files before repo `.env` and shell overrides.
- Added warnings for dangling current profiles and for active-profile overrides from `.aiwf/metadata.local.env` or shell metadata variables.
- Documented metadata precedence, profile/local override behavior, and the runtime-option boundary.

## Compatibility

- `WORKFLOW_PROTOCOL_VERSION` remains `1.7.5`.
- No event schema change.
- No task front matter schema change.
- No AI attribution metadata precedence change.
- No historical migration.
