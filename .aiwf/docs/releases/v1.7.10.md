# AIWF v1.7.10

## Release Type

Runtime security hardening release for filesystem trust-boundary handling.

## Summary

AIWF v1.7.10 hardens runtime filesystem handling across public export,
review bundle generation, and upgrade source validation.

This release is based on:

- `a644c14` Fix P0 symlink handling in export bundle and upgrade
- `c76cc55` Add shared safe path guards for AIWF filesystem operations

## Changed

- Public export now rejects symlink traversal under allowlisted roots.
- Review bundle generation now rejects symlinked task content.
- Upgrade source validation now rejects symlinked source-package members.
- Review bundle output now refuses protected workflow evidence paths.
- Existing review bundle output files now require explicit `--force`.

## Compatibility

- Normal public export, review bundle, and upgrade command syntax remains unchanged.
- Only review bundle overwrite behavior changed: overwriting an existing output now requires `--force`.
- No workflow protocol semantic change.
- No event schema change.
- No phase state machine change.

## Version Identity

- release version: `1.7.10`
- tool version: `1.7.10`
- workflow protocol version: `1.7.8`

## Boundary

This is a v1.7 runtime security hardening release.
It does not introduce v1.8 workflow semantics, new event schemas, or task lifecycle changes.
