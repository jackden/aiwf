# AIWF v1.7.5.post4

## Release Type

Metadata init interactive help hardening for v1.7.5.

## Changed

- Added inline allowed-values help to `./aiwf metadata init` prompts.
- Supported current-field help tokens: `?`, `:list`, `:help`, `L`, and `l`.
- Supported `:all` during `metadata init` to print the full metadata allowed-values reference and then re-prompt the same field.
- Reused the existing metadata allowed-values registry so prompt-time help stays aligned with `./aiwf metadata allowed-values`.
- Documented inline metadata-init help in operator-facing docs.
- Bumped `AIWF_TOOL_VERSION` to `1.7.5.post4` so emitted provenance matches the shipped usability patch.

## Compatibility

- `WORKFLOW_PROTOCOL_VERSION` remains `1.7.5`.
- No workflow protocol semantic change.
- No event schema change.
- No metadata precedence change.
- No profile create/use semantic change.
- No metadata migration.
