# AIWF v1.7.0 Release Note

## Positioning
v1.7.0 is the Pre-edit Governance MVP release.

It aligns protocol and release wording with the first task-level pre-edit guard while keeping finalize semantics unchanged.

## Included Work
- `001_aiwf_agents_md_execution_gate_simplification`
- `002_aiwf_agents_md_final_response_aiwf_state_fix`
- `003_aiwf_task_level_pre_edit_guard`

## Added
- `./aiwf guard --pre-edit --path <task_path>`
- stable guard diagnostics:
  - `AIWF-GUARD-PASS`
  - `AIWF-GUARD-001`
  - `AIWF-GUARD-002`
  - `AIWF-GUARD-003`
  - `AIWF-GUARD-004`
  - `AIWF-GUARD-900`

## Changed
- protocol wording now reflects v1.7.0 pre-edit governance semantics
- README release baseline now points to v1.7.0
- the release note set now names the guard MVP boundary explicitly

## Non-Goals
- repo-level dirty tree detection
- automatic active task detection
- task_created event bootstrap
- retrospective_bootstrap / detected_by
- Codex skill / Claude hook / pre-commit hook
- policy-on-path
- strict phase state machine

## Compatibility
- finalize semantics are unchanged
- check / report / export semantics are unchanged
- `guard --pre-edit` is a task-level gate only; it does not guarantee repo-level workflow enforcement
