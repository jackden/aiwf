# AIWF v1.7.12

## Release Type

Runtime safety and integrity hardening.

## Summary

This unreleased baseline consolidates the F-01 through F-08 CLI semantic
hardening work. It narrows repository-boundary handling, preserves evidence
integrity, and makes deterministic user-input rejection fail closed without
changing the workflow protocol or event schema.

Version identity:

- release version: 1.7.12
- tool version: 1.7.12
- workflow protocol version: 1.7.8

## Repository Boundary Hardening

- agents check --path and agents install --path require the resolved target to
  remain within the repository root. Absolute outside paths, parent traversal,
  final symlink escapes, and parent-directory symlink escapes fail closed with
  AIWF-AGENTS-PATH-001 and return code 2.
- The --yes option only accepts the agents-install confirmation. It does not
  bypass repository-boundary validation.
- Dataset export rejects outputs under the configured AIWF records root with
  AIWF-DATASET-OUTPUT-001, including nested and symlink-resolved paths.
- Dataset export distinguishes an output that resolves outside the repository
  with AIWF-DATASET-OUTPUT-002. Valid repository-local outputs outside records
  remain supported.
- Shared command target paths reject missing targets with AIWF-CLI-PATH-001
  and repository escapes with AIWF-CLI-PATH-002 before mutation.

## Migration and Upgrade Integrity

- relocate --apply --legacy-docs fails closed before mutation when a legacy
  source and canonical destination both exist.
- upgrade --apply --migrate-legacy-docs uses the same conflict preflight and
  does not emit success evidence when blocked.
- Relocation conflicts return code 2 with AIWF-RELOCATE-CONFLICT-001.
  Already-relocated entries remain truthful no-ops; no overwrite, merge, or
  delete behavior was added.

## Task Creation and Backfill Integrity

- next-id and internal task-ID lookup are read-only. A missing AI date returns
  001 without creating a date directory, index, event, or report.
- A conflicting non-directory date path fails closed with AIWF-TASK-ID-001.
- new-task validates deterministic metadata before allocating task identity,
  preventing invalid input from leaving orphan date directories or partial
  artifacts while preserving valid max-plus-one creation.
- Backfill persists deterministic source provenance, returns a no-op for a
  complete matching rerun, and fails closed for different or ambiguous
  same-name source identities.
- backfill --update-existing only fills missing artifacts for matching
  provenance. It does not overwrite historical task, validation, review,
  index, event, or finalized evidence.

## Dataset and Read-only Command Safety

- finalize --dry-run and check --finalize-ready do not append workflow events
  or create missing event-log paths.
- Normal finalize still emits finalize evidence only after successful
  metadata/index mutation. Readiness output is not closure evidence.
- Dataset boundary rejection occurs before collection or output replacement.

## Selector and Diagnostic Consistency

- Package records and task listing validate status, workflow phase, and review
  status selectors exactly and case-sensitively. Invalid selectors fail before
  discovery or output mutation; valid zero-match selectors still succeed.
- package records rejects reversed date ranges before discovery or output
  mutation.
- New-task task references, tags, and related files use distinct diagnostics:
  AIWF-META-REF-001, AIWF-META-TAG-001, and AIWF-META-011.
- Deterministic user-input, selector, path, and confirmation validation
  failures addressed by this release return code 2. This does not redefine
  exit behavior for configuration failures, internal invariants, unexpected
  runtime failures, argparse usage errors, or interruption.

## Compatibility

- No event schema, manifest schema, workflow phase, finalize semantic, or
  workflow protocol change was made.
- Existing legal repository-local agents paths, valid new-task creation,
  valid repository-local dataset outputs, and valid zero-match selectors
  remain supported.
- No lock, reservation, transaction-directory, or new rollback subsystem was
  introduced.

## Deferred

- Publication actions (commit, push, tag, and GitHub Release) are intentionally
  outside this release-consolidation work.
- The release-consolidation runner and matrix are temporary validation assets;
  they are not publication artifacts.

## Known Limitations

- The current doctor CLI intentionally supports task-specific paths only; this
  release does not introduce repository-level doctor semantics.
- Public test source contains the synthetic fixture
  .aiwf/records/ai_20260718/dataset.json. It is allowlisted because it is
  test-only, contains no task record or event payload, and has no private path
  or credential.
