# AIWF v1.7.5

## Release Type

Deterministic metadata hardening.

## Changed

- Expanded `new-task` with structured metadata inputs.
- Added canonical task reference handling for `parent_task`, `related_tasks`, `blocked_by`, and `supersedes`.
- Added strict metadata list item validation.
- Added repo-relative path validation for `related_files`.
- Added non-today date guard for `new-task`.
- Ensured malformed metadata blocks `check` and `finalize` through existing diagnostics.
- Clarified that task front matter is tool-owned workflow metadata.
- Defaulted new task owner/reviewer roles to `ai-agent` / `human` instead of Codex-specific attribution.
- Preserved historical `owner: "codex"` / `reviewer: "codex"` records as compatible evidence rather than migrating or rejecting them.
- Clarified that actual tool/provider/model provenance stays in metadata profiles and event records, not task front matter role fields.

## Compatibility

- No event schema change.
- No strict phase state machine.
- No workflow orchestration behavior.
- No historical record migration.
- No review content quality blocker.

## Deferred

- Full metadata edit command.
- Review artifact content quality blocker.
