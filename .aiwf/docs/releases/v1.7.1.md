# AIWF v1.7.1

## Release Type
Patch-level documentation, packaging, and reviewability hardening.

## Baseline
Built on v1.7.0 baseline.

## Included
- Clarified packaging boundary for repository snapshots.
- Documented that `.env` must not be included in shared zip artifacts.
- Clarified task naming convention for future tasks.
- Clarified that `guard --pre-edit` is a pre-edit governance guard, not finalize.
- Improved reviewer-facing explanation of v1.7.0 behavior.
- Aligned emitted event `tool_version` metadata with the v1.7.1 tool baseline for newly generated events.
- Clarified finalized-artifact immutability and follow-up errata workflow boundaries.
- Corrected `AIWF-PATH-015` recovery wording to match current runtime behavior (`finalize` idempotent/no-op on closed tasks).

## Runtime Behavior
Metadata provenance fix and governance wording hardening only. No workflow semantic change for finalize, check, doctor, guard, event schema behavior, or completion gate semantics.

## Non-goals
- No strict phase state machine.
- No amend-finalized runtime command.
- No reopen runtime command.
- No manifest rewrite mechanism.
- No policy-on-path.
- No hook integration.
- No automatic task detection.
- No finalized task path migration.
